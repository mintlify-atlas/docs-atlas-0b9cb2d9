April 14 crystallized a theme that BuilderPulse had been tracking for weeks: the most valuable developer assets in the AI era aren't necessarily code. Alongside that signal, a founder running multiple $10K MRR companies on a $20/month tech stack described getting rejected from VC pitch nights because investors couldn't understand why he'd need their money.

## Build recommendation

**2-hour project: Claude Code Skills Marketplace / Discovery Tool**

Build a simple web app that indexes public GitHub repos containing CLAUDE.md or `.claude/skills/` directories, lets users browse, search, and rate them, and generates a one-click install command. You can build the indexer with the GitHub API and ship the frontend as a static site.

Monetization: featured listings for commercial skill authors, or a $5/month "pro skills" subscription that unlocks private skill uploads and team sharing.

## Counter-intuitive findings

**A Polymarket bot that always bets "No" on everything is profitable.** [Nothing Ever Happens](https://github.com/sterlingcrispin/nothing-ever-happens) (395 points, 214 comments) automatically buys "No" on every non-sports prediction market. The thesis: prediction markets have a systematic bias toward dramatic outcomes because bettors are drawn to exciting "Yes" scenarios. The boring outcome — nothing happens — is consistently underpriced.

The deeper insight for builders: the default outcome in most markets is stasis. If you're building prediction features, recommendation engines, or trend detectors, calibrate against a "nothing happens" baseline. Most products over-index on novelty.

**VCs keep rejecting a founder who runs multiple $10K MRR companies.** The rejection feedback from pitch nights: "What do you even need funding for?" The extreme lean bootstrapper enters a paradox — margins too high to attract investment, because VCs invest in growth potential, not current profit. 940 points and 499 comments show this resonates deeply with indie developers.

**WordPress plugin trust is structurally broken.** @cold_tom: "The scariest part isn't the backdoor itself, it's how normal the acquisition looked. Buying a trusted plugin and pushing an update is basically indistinguishable from legitimate maintenance." The FAIR package manager project tried to fix this decentrally and recently shut down — the gap remains open.
