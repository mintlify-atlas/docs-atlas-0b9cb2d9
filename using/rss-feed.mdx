BuilderPulse publishes every daily report as a commit to its GitHub repository. GitHub exposes a standard Atom feed for every repository's commit history, which means you can subscribe to new BuilderPulse reports in any RSS reader — no special account or API key required. The feed updates the moment a new report is pushed.

**Feed URL:**

```text
https://github.com/BuilderPulse/BuilderPulse/commits/main.atom
```

## What each feed item contains

Each item in the Atom feed corresponds to one commit. For BuilderPulse, that means one daily report per item. Here is what you'll see in your reader:

- **Title** — the commit message, which contains the report date and the day's top headline (for example: `BuilderPulse Daily — April 16, 2026`)
- **Link** — points to the commit on GitHub, from which you can navigate directly to the `.md` report file
- **Date** — the timestamp of the commit in Shanghai time
- **Author** — Liu Xiaopai (刘小排), the author of BuilderPulse

<Note>
  The commit link leads to the GitHub commit page, not directly to the `.md` file. From there, click on the file name in the changed files list (for example `en/2026/2026-04-16.md`) to open the rendered report.
</Note>

        Click **+ Add content** in the left sidebar.

## Telling English and Chinese commits apart

The Atom feed includes commits for both the English edition (`en/`) and the Chinese edition (`zh/`). In practice, both editions are committed together as a single daily commit, so you will see one feed item per day covering both.

If you inspect the commit message or the changed files in a given commit, you can identify which report is which by the file path:

- English reports: `en/YYYY/YYYY-MM-DD.md`
- Chinese reports: `zh/YYYY/YYYY-MM-DD.md`

<Tip>
  The Chinese edition report file always begins with Chinese characters in the title (`# BuilderPulse Daily — 2026 年 4 月 16 日`), making it easy to distinguish at a glance once you open the file.
</Tip>

## Organizing your reading workflow

A few tips for keeping the daily reports useful without creating noise:

- **Create a dedicated folder.** In any reader, put the BuilderPulse feed in its own folder or collection so it doesn't get buried in higher-volume feeds.
- **Mark as read on open.** Configure your reader to automatically mark items as read when you open them. Since BuilderPulse publishes once per day, there's no risk of losing a report by marking it read.
- **Use the commit link as a permalink.** Each feed item links to a stable GitHub commit URL. Bookmark individual reports for later reference using the commit link — it won't change.
- **Skim the headline, read the Action section.** Each report's top-of-file summary and the "What should I build?" section give you the high-signal takeaway in under two minutes.
